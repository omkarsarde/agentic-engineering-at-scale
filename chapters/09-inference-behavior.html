9.3 One request has two latency phases

A user sends a 2,000-token contract and asks for a 100-token answer. The first visible token arrives slowly. The remaining tokens stream at a steadier cadence. A dashboard reports one latency number, so an engineer shortens the output limit. Total latency falls, but the long initial pause remains. The intervention changed the wrong phase.

Autoregressive inference has two mechanically different phases. During prefill, the model processes the prompt positions in parallel and constructs a key–value cache for every layer. During decode, it produces one new position, appends that position’s keys and values to the cache, and repeats. Prefill can expose large matrix multiplications to the accelerator. Decode has a serial dependency: token t+1 cannot be chosen before token t becomes context.

Three request-level measurements keep the distinction visible:

  • Time to first token (TTFT) starts when the caller submits a request and ends when the first output token is available. End-to-end TTFT can include network transit, queueing, tokenization, prefill, and the first decode decision. A model-only TTFT excludes some of those terms, so every report must state its boundary.
  • Time per output token (TPOT) is the average time between output tokens after the first. Inter-token latency (ITL) is each individual gap; its tail exposes stalls hidden by a mean.
  • End-to-end latency for n_{out} output tokens is approximately

L \approx \mathrm{TTFT} + (n_{out}-1)\,\mathrm{TPOT}.

The approximation is useful only when accompanied by its units and scope. Stop detection, detokenization, network flushing, variable step time as the cache grows, and scheduling can make individual gaps differ. Still, it immediately explains why shortening a response mostly affects the second term and shortening a long prompt mostly affects the first.

Figure Figure 9.1 asks where a millisecond can enter one request. It deliberately ends at the boundary of a single request. Continuous batching, queue disciplines, prefix caching, speculative decoding, goodput, and production service objectives belong to Chapter 10.

The durable measurement rule is to preserve both phases. A single percentile for total latency cannot tell whether prompt work, serial generation, queueing, or streaming is responsible.

9.5 KV bytes place the phases on a roofline

The KV cache prevents decode from recomputing every earlier key and value. This chapter needs one result from that mechanism: its storage grows linearly with sequence length. For a conventional decoder cache,

M_{KV}=2LBTH_{KV}d_hs \quad \text{bytes}.

Here L is the number of layers, B the batch size, T the cached sequence length, H_{KV} the number of key/value heads, d_h the head dimension, and s the bytes per stored scalar. The leading 2 accounts for keys and values. Query heads do not appear. In multi-head attention, H_{KV} usually equals the number of query heads. Grouped-query attention uses fewer KV heads, and multi-query attention uses one. A latent-attention architecture stores a compressed representation with architecture-specific additions, so substituting its query-head count into this formula is wrong.

Attention cache form Width term Cache ratio against H_Q-head MHA
Multi-head attention H_{KV}=H_Q 1
Grouped-query attention H_Q > H_{KV} > 1 H_{KV}/H_Q
Multi-query attention H_{KV}=1 H_Q^{-1}
Latent/compressed attention compressed latent plus model-specific state derive from that architecture

The fixture uses 32 layers, eight KV heads, head dimension 128, batch one, and two-byte cache values. It therefore stores 128 KiB per cached token: 16 MiB at 128 tokens, 64 MiB at 512, and 256 MiB at 2,048. These are configuration arithmetic, not measurements of a named model.

Storage explains growth; the roofline model explains why prefill and decode often hit different hardware limits. Arithmetic intensity is

I=\frac{\text{floating-point operations}}{\text{bytes moved from the limiting memory level}}.

If hardware sustains peak compute P_{max} FLOP/s and memory bandwidth W_{max} byte/s, attainable performance is bounded by

P(I)\leq \min(P_{max},\;I W_{max}).

The ridge point I^*=P_{max}/W_{max} separates the bandwidth-limited slope from the compute ceiling. For a weight matrix W\in\mathbb{R}^{m\times k} multiplied by b activation vectors, the dominant work is about (2mkb) FLOPs. If the weights dominate traffic and each weight occupies s bytes, intensity is approximately (2b/s). With two-byte weights, I\approx b. A decode matrix multiplication has a small effective b; prefill exposes many prompt positions and has a much larger one.

This is a placement model, not a latency oracle. Attention reads growing KV state. Fused kernels change traffic. Caches above HBM, quantization, sparsity, and tensor parallel communication add ceilings. The fixture intentionally uses an illustrative 80 TFLOP/s compute ceiling and 2 TB/s memory ceiling, giving a ridge at 40 FLOP/byte. Its decode point at intensity one is bandwidth-bound; all three prefill points are compute-bound. Chapter 10 turns these corollaries into batching and serving decisions.

# code/ch09/request_profile.py — changed region: request_profiles
def kv_bytes(tokens: int, batch: int = 1) -> int:
    return (2 * MODEL["layers"] * MODEL["kv_heads"] * MODEL["head_dim"]
            * tokens * batch * HARDWARE["dtype_bytes"])

def request_profiles(output_tokens: int = 128):
    parameter_bytes = MODEL["parameters"] * HARDWARE["dtype_bytes"]
    for tokens in (128, 512, 2048):
        prefill_ms = max(2 * MODEL["parameters"] * tokens / HARDWARE["peak_flops"],
                         parameter_bytes / HARDWARE["memory_bandwidth"]) * 1000
        decode_bytes = parameter_bytes + kv_bytes(tokens)
        tpot_ms = max(2 * MODEL["parameters"] / HARDWARE["peak_flops"],
                      decode_bytes / HARDWARE["memory_bandwidth"]) * 1000
        yield prefill_ms + tpot_ms, tpot_ms, kv_bytes(tokens)

9.6 Decoding turns logits into a choice

At output position t, the model produces one logit z_i for every vocabulary token i. A decoding policy turns that vector into a token. Greedy decoding returns an index with maximum logit. Sampling first defines probabilities and draws from them. Both choices are caller policy layered over the same model output.

Temperature rescales logits before softmax:

p_i(T)=\frac{\exp(z_i/T)}{\sum_j\exp(z_j/T)}, \qquad T>0.

Equivalently, if p_i(1) is the ordinary softmax probability, then p_i(T) is proportional to p_i(1)^{1/T}. Lower T sharpens the distribution; higher T flattens it. Dividing all logits by the same positive value preserves their order. Temperature changes gaps and entropy, not which token has the highest logit.

Implement the small-T limit as a branch to argmax rather than evaluating exp(z/T) with a near-zero divisor. Subtracting the maximum scaled logit before exponentiation prevents overflow without changing probabilities. The fixture’s softmax does both. At temperature=0, it returns a one-hot greedy distribution with deterministic tie-breaking by index. That makes the choice rule deterministic for a fixed logit vector. Section Section 9.9 will show why the vector itself may still move.

Greedy decoding optimizes one local decision at each step, not the probability or usefulness of a complete sequence. Beam search retains several high-scoring prefixes and usually normalizes sequence scores to reduce a short-output bias. It is search, not stochastic sampling; maximizing likelihood can also produce generic or repetitive text. Chapter 8 owns search and verifier-driven selection. This chapter keeps beam search as an exercise so the sampler remains inspectable.

The bounded Route-B backfill for Chapter 12 begins here. A reader who understands the map logits → policy → token, the temperature equation, and the determinism contract in Section Section 9.9 has the sampling foundation needed to call a model API without treating its parameters as magic.

9.7 Truncation shapes the candidate support

Sampling from the full softmax preserves every token with nonzero probability. A large vocabulary can leave a long tail of locally implausible tokens. Truncation masks part of the distribution and renormalizes the survivors before drawing. It is a bias–diversity decision: remove too little and tail events can damage coherence; remove too much and useful alternatives disappear.

Three policies express different invariants:

  • Top-k retains exactly the k highest-probability tokens, except that a defensive implementation always retains at least one. Its support size is fixed even when the distribution is flat or sharply peaked.
  • Nucleus or top-p sorts tokens by descending probability and retains the smallest prefix whose cumulative mass reaches p. Its support adapts to distribution shape. The token that crosses the threshold belongs in the set; applying the mask before including it creates an off-by-one error. For any threshold, the top token keeps the set nonempty.
  • Min-p retains token i when p_i\geq \alpha p_{max}, where p_{max} is the largest probability and alpha is a relative cutoff. Its threshold rises and falls with the leading token. This can preserve more candidates at high temperature than a fixed cumulative-mass rule while still removing probabilities small relative to the mode.

Locally typical sampling offers another view: retain tokens whose information content -\log p_i lies near the distribution’s entropy, then stop when their mass reaches a budget. It can reduce degenerate repetition on some open-ended tasks, but no truncation policy dominates across workloads.

The order temperature → probabilities → truncation → renormalization matters because top-p, min-p, and typicality inspect probabilities after temperature has reshaped them. On the fixture vector at temperature 1.2, full sampling retains five tokens, top-2 retains two, top-p at 0.8 retains three, min-p at 0.15 retains four, and typical sampling at 0.8 retains three. These support counts are local facts about one vector, not universal recommended settings.

9.8 Processor order changes the policy

Real generation applies more than temperature and a support mask. A logit processor changes logits using token history or caller policy. A logit bias adds a token-specific offset. A presence penalty acts once after a token has appeared. A frequency penalty grows with its count. A repetition penalty often uses a sign-aware multiplicative rule: divide a positive seen-token logit by r>1, but multiply a negative one by r. Applying the same division to negative logits would make them less negative and accidentally reward repetition.

Let c_i be the count of token i, a_i an explicit bias, f a frequency penalty, and u a presence penalty. One additive form is

z'_i=z_i+a_i-fc_i-u\mathbf{1}[c_i>0].

If the result is then divided by temperature, an additive offset’s effective log-probability strength scales as (1/T). Moving the same penalty after temperature changes the policy. Hard grammar masks, banned tokens, forced tokens, EOS rules, and provider-specific processors add further ordering choices. Record the effective pipeline, not merely parameter names.

Figure Figure 9.2 shows the conceptual order used by the fixture. A serving library may combine stages or use a documented different order; that variation belongs in a versioned adapter test.

Penalties are behavioral tools, not semantic constraints. Source code may need the same closing bracket, variable name, or indentation token many times. Aggressive repetition suppression can make valid syntax unreachable. Logit bias can ban a surface token while a synonym or alternate tokenization expresses the same content. If a property must hold, use a parser, schema, or deterministic validator at the appropriate boundary; Chapter 10 covers production structured decoding.

9.9 Seeded is not batch invariant

Set temperature to zero, fix a seed, send the same prompt twice, and receive different text under load. Nothing requires sampling to have caused the difference. Greedy output changes if a tiny numerical difference changes the ordering of two nearly tied logits.

Floating-point addition is not associative. For finite-precision values, (a+b)+c may differ from a+(b+c). GPU kernels partition reductions across threads and tiles. A runtime may choose a different kernel or split when batch shape, sequence packing, quantization path, or hardware changes. Those legal execution paths can produce slightly different logits. Once an argmax flips, the new token changes every later autoregressive input, so a minute drift can become a different completion.

A seed controls draws from pseudorandom generators. It does not pin weights, tokenizer revision, prompt serialization, kernel selection, reduction order, request batching, model replica, quantization, or postprocessing. Use four distinct reproducibility claims:

  1. Seed replay: the same probability vectors and random-number stream yield the same sampled indices.
  2. Run-to-run numerical repeatability: the same request and execution environment yield the same intermediate values.
  3. Batch invariance: one request yields the same result regardless of which other requests share the batch or how its sequence is partitioned.
  4. Cross-environment bitwise identity: results remain identical across hardware, driver, library, model, and tokenizer revisions. This is the strongest and least commonly available claim.

The lab’s narrated failure is intentionally small. Token A’s logit contribution is reduced from [1e16, 1, -1e16, 0]. Serial summation loses the 1 and returns zero. A partitioned order cancels the large terms first and returns one. Token B stays at 0.5. The two legal orders flip the greedy winner and move the maximum probability by 0.244919. This is a pedagogical construction, not a measured estimate of drift in a real model.

# code/ch09/request_profile.py — changed region: batch_invariance_probe
parts = [1e16, 1.0, -1e16, 0.0]
serial = sum(parts)
partitioned = (parts[0] + parts[2]) + (parts[1] + parts[3])
single = softmax([serial, 0.5])
batched = softmax([partitioned, 0.5])
greedy_flip = single.index(max(single)) != batched.index(max(batched))
max_drift = max(abs(a - b) for a, b in zip(single, batched))

9.10 Hallucination is a task-relative failure

Now consider a different failure. A user asks, “Which treaty created the Aerolith Maritime Council in 1987?” The entity is fabricated. A model produces a fluent treaty name, date, membership list, and citation. Lowering temperature makes the same invention appear more consistently. The inference loop is behaving as configured; the answer fails against the world.

Hallucination is useful only after the evaluation contract says what the output should be faithful to. The term otherwise collapses different defects with different remedies:

Failure Reference violated Example Appropriate first measurement
Factual error external world state wrong capital or date atomic factual correctness
Fabrication existence in the world or source set invented entity or paper existence/provenance check
Unsupported claim supplied evidence true-sounding claim absent from sources claim-level support
Source contradiction supplied evidence answer reverses a policy clause entailment/contradiction
Stale knowledge required snapshot time names an earlier office holder time-stamped correctness
Task misunderstanding user intent or schema answers a related question intent/task success
Unfaithful explanation claimed causal reasoning rationale does not support answer process/faithfulness study

Long-form factuality should be decomposed into atomic claims before scoring; one correct paragraph can contain one consequential false sentence. A citation is not evidence until the cited source exists, is authoritative for the claim, and entails it. Core retrieval, source selection, citation entailment, and attribution mechanics belong to Chapter 14. This chapter owns the prior decision: whether the system has enough calibrated support to answer at all.

Why does the failure occur? Next-token training rewards predictions that match text distributions. A false premise can resemble a legitimate historical question, and likely continuations include treaty vocabulary. Finite training data, ambiguous or conflicting sources, model capacity, distribution shift, and lossy parametric storage all create error. Post-training can reward helpful, decisive answers. Evaluations that give zero for abstention but some chance of credit for guessing reinforce the same behavior. The 2025 analysis Why Language Models Hallucinate formalizes this guessing incentive; TruthfulQA earlier demonstrated that imitating human text can reproduce popular false beliefs.

No decoding rule repairs a knowledge gap in general. Greedy decoding can select the most probable false continuation. Sampling may occasionally find a correct answer but also expands the tail. Self-consistency can concentrate a shared misconception. A model’s critique can share its first answer’s missing knowledge. External evidence and independent checks can change the information state; abstention can prevent an unsupported guess from being presented as fact.

This section begins the bounded Chapter 14 backfill cluster. The durable sequence is: define the reference, decompose the failure, measure a confidence signal against correctness, and enforce an action below threshold. “Hallucination rate” without those choices is not a reproducible metric.

9.11 Confidence must forecast correctness

A confidence score q(x) is calibrated for a binary event Y when cases assigned confidence near c are correct about a fraction c of the time:

\Pr(Y=1\mid q(X)\approx c)\approx c.

Calibration is population- and event-specific. A score calibrated for exact short-answer correctness may not be calibrated for every atomic claim in a long response, for policy compliance, or after a domain shift.

Token log probability is one possible signal, not a probability that a world claim is true. Summing token log probabilities gives a sequence log probability, which usually becomes more negative with length. Averaging removes that direct length scaling but changes the quantity and can favor verbose continuations differently. A probability assigned to “True” after showing the model its own answer, often called P(True), asks a more relevant event; Kadavath et al. found useful calibration in studied settings and weaker cross-task generalization for some self-knowledge signals. Verbalized confidence can outperform raw conditional probabilities for some post-trained models, as Tian et al. reported, but it remains a learned utterance that must be calibrated on the deployment task.

Open-ended answers introduce a symmetry: “Paris” and “The capital is Paris” are different strings with the same meaning. Surface-form entropy counts them separately. Semantic entropy first groups mutually equivalent answers into meaning clusters c, sums probability mass inside each cluster, and computes

P(c\mid x)=\sum_{y\in c}p(y\mid x), \qquad H_{sem}(x)=-\sum_c P(c\mid x)\log P(c\mid x).

In the fixture, surface probabilities [0.36, 0.24, 0.20, 0.12, 0.08] have entropy 1.489 nats. The first two are paraphrases, so merging them lowers semantic entropy to 1.085 nats. Real semantic-entropy systems sample responses and use bidirectional entailment or another semantic equivalence model. The clustering model can make mistakes, the samples are correlated, and entropy may reflect legitimate ambiguity rather than ignorance. Kuhn, Gal, and Farquhar introduced the linguistic-invariance formulation; later work demonstrated hallucination detection with semantic entropy in open-ended generation.

Calibration needs a loss, not only a diagram. For binary correctness y\in\{0,1\} and score q:

\mathrm{Brier}(q,y)=(q-y)^2,

and log loss is -y\log q-(1-y)\log(1-q). Both are proper scoring rules: in expectation, truthful probabilities minimize the score. Expected calibration error (ECE) bins predictions, compares mean confidence with bin accuracy, and weights the gaps. ECE is readable but depends on bin boundaries and sample size; report a proper score and a reliability diagram with it.

The lab fits one scalar temperature on a calibration split, then evaluates a disjoint deterministic fixture. It moves ECE from 0.060 to 0.024 and Brier score from 0.2015 to 0.1983. Temperature calibration is monotone, so it changes the meaning of a threshold but not the ranking of examples. A risk–coverage curve and its area can therefore remain unchanged even when ECE improves. Calibration measures probability fidelity; discrimination measures whether safer cases rank above riskier ones.

9.12 Abstention spends coverage to buy reliability

An abstention policy maps uncertainty to an action. Below a threshold, the system might say it lacks support, ask a clarifying question, retrieve evidence, narrow the claim, route to a qualified reviewer, or refuse a request. The model may propose a confidence score; deterministic caller code must enforce the threshold and record the branch.

Let g_t(x)=\mathbf{1}[q(x)\geq t] mean “answer at threshold t,” and let loss \ell(x,y) be one for a wrong answer and zero for a correct one. Coverage is the fraction answered,

C(t)=\mathbb{E}[g_t(X)].

Two quantities are often both called risk:

R_{selective}(t)=\frac{\mathbb{E}[\ell g_t]}{\mathbb{E}[g_t]}, \qquad R_{marginal}(t)=\mathbb{E}[\ell g_t].

Selective risk is the error rate among answered queries. Marginal error is the fraction of all incoming queries that receive a wrong answer; abstentions count as zero loss. They differ by coverage: R_{marginal}=C\,R_{selective}. A requirement saying “ten percent risk” is incomplete until it names one.

Sweeping t creates a risk–coverage curve. A useful confidence ranking lowers selective risk as coverage falls. Area under the risk–coverage curve (AURC) summarizes that ranking, but important operating regions and slices should still be reported. Chapter 22 owns interval estimates and release comparisons; here the curve is the mechanism that turns a threshold into an explicit trade.

Split conformal prediction adds a finite-sample coverage statement without assuming that raw model probabilities are calibrated. On a held-out calibration set, define nonconformity s_i=1-p_i(y_i) using the probability assigned to the true answer. Sort the n scores and choose the order statistic at rank \lceil(n+1)(1-\alpha)\rceil. For a new input, return all answers whose nonconformity is at most that threshold. Under exchangeability of calibration and future examples, the set contains the true answer with marginal probability at least (1-\alpha).

One safe interface answers only when this conformal set is a singleton and abstains otherwise. On the synthetic fixture with \alpha=0.10, the threshold is 0.8625. Test set coverage of the true label is 0.93; 0.66 of queries produce singleton sets; singleton error is 0.106. The guarantee concerns true-label inclusion in the set. It does not guarantee that each singleton is correct or that the conditional error among answered cases is below ten percent.

Conformal risk control (CRC) extends the idea to a bounded loss that changes monotonically with a threshold. For the lab’s marginal wrong-answer loss, a simple binary correction accepts a threshold only when

\frac{e_t+1}{n+1}\leq \alpha,

where e_t is the number of wrong answers that the threshold would release on n calibration examples. Choose the lowest accepted threshold to maximize coverage. Under the method’s exchangeability and loss assumptions, this controls expected marginal loss up to the conformal correction. With \alpha=0.10, the fixture selects confidence 0.85, answers 40 percent of queries, and incurs marginal error 0.07. Yet its selective risk is 0.175. The build is designed to make that distinction impossible to miss.

# code/ch09/selective_prediction.py — changed region: crc_threshold
def crc_threshold(items, alpha):
    for threshold in sorted({item["confidence"] for item in items}):
        errors = sum(item["confidence"] >= threshold and not item["correct"]
                     for item in items)
        if (errors + 1) / (len(items) + 1) <= alpha:
            return threshold
    return math.inf

A conformal guarantee is not permanent certification. Exchangeability can fail when users, prompts, languages, evidence availability, or models change. Calibration data can be contaminated or too small for critical slices. A learned confidence model can be attacked. Monitor coverage and outcomes, freeze the policy version used for each decision, and fall back to a conservative route when drift invalidates the evidence. For consequential applications, also price the cost of abstention: delayed care, inaccessible service, and overloaded human queues are real harms.

9.13 Streaming tokens are not text chunks

An inference server emits token IDs. A user interface displays Unicode text. Those boundaries need not align. UTF-8 represents one Unicode scalar value with one to four bytes, and byte-level tokenizers can expose a piece that ends inside a multibyte character. Decoding each event independently may emit replacement characters or raise an error. A token can also decode to an empty string until combined with neighbors.

Stop sequences create the same boundary problem at a different level. If the requested stop is STOP, one chunk may end in ST and the next begin with OP. Checking each chunk separately leaks the stop marker and may stream text that should have remained hidden. Buffer at least the longest possible stop-prefix suffix, run an incremental decoder, and emit only text that can no longer participate in a stop.

# code/ch09/decoding_runtime.py — changed region: stream_until
decoder = codecs.getincrementaldecoder("utf-8")()
pending, emitted = "", []
for chunk in chunks:
    pending += decoder.decode(chunk)
    if stop in pending:
        emitted.append(pending.split(stop, 1)[0])
        return "".join(emitted)
    keep = max(0, len(stop) - 1)
    if len(pending) > keep:
        emitted.append(pending[:-keep] if keep else pending)
        pending = pending[-keep:] if keep else ""

The lab splits the UTF-8 bytes for 🙂 across three events and splits STOP across two. The emitted text is exactly A🙂; neither a replacement character nor stop text leaks. A production implementation must additionally handle several stop strings, normalization policy, end-of-sequence tokens, cancellation, maximum-token termination, client disconnects, and an explicit finish reason. Never infer “the model chose to finish” when the server actually hit a length limit.

This edge is where byte-level reproducibility meets user-visible behavior. Comparing only token IDs misses detokenizer and stop-policy changes. Comparing only final text misses token-score drift and hidden stop leakage. Preserve both representations when debugging.

9.16 What endures, what changes

What endures. Prefill and decode create different latency terms. KV storage depends on KV heads and grows linearly with cached tokens. Roofline placement follows arithmetic intensity. Temperature rescales probability gaps without reordering logits; truncation changes support; processor order changes the final policy. A seed controls randomness, not every numerical or scheduling path. Hallucination is defined against a reference and task. Token likelihood, semantic uncertainty, calibration, discrimination, set coverage, marginal error, and selective risk are different quantities. Abstention is a caller-enforced action with an explicit coverage cost. Token events require buffered detokenization and stop handling.

What changes. Model and tokenizer revisions, cache layouts, accelerator ceilings, kernel selection, deterministic-mode support, sampler names/defaults, processor order, log-probability APIs, uncertainty estimators, benchmark results, and provider guarantees all move. Keep those facts in adapters, measurement metadata, and the dated landscape. Removing the dated block leaves the request, decoding, calibration, and abstention mechanisms intact.
